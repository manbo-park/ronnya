# 점수 계산기 로직 · 제약 정리

`src/features/Calculator.tsx`(화면)와 `src/core/fu.ts`·`src/core/points.ts`(순수 로직)로
구성된 **점수 계산기**(홈 메뉴 "계산하기")의 동작과 제약을 정리한다. 도라·역 판정을 자동으로
하지 않고 **사용자가 판수를 직접 입력**해 부수·점수만 계산하는 도구라는 점이 점수 계산 연습
([score-trainer-plan.md](score-trainer-plan.md))과의 핵심 차이다.

룰 기준과 의도적 차이(뒷도라·키리아게 만관 미적용 등)는 [../CLAUDE.md](../CLAUDE.md)를 따른다.

---

## 1. 입력 모델

부수 계산 입력(`FuInput`, `core/fu.ts`):

| 필드      | 값                                                                                                |
| --------- | ------------------------------------------------------------------------------------------------- |
| `special` | `none` / `pinfu`(핑후) / `chiitoi`(치또이)                                                        |
| `winForm` | `tsumo`(쯔모) / `menzenRon`(멘젠 론) / `furoRon`(후로 론)                                         |
| `wait`    | `ryanmen`(양면) / `kanchan`(간짱) / `penchan`(변짱) / `tanki`(단기) / `shanpon`(샤보)             |
| `pair`    | `normal`(수패·객풍) / `yakuhai`(역패) / `roundWind`(장풍) / `seatWind`(자풍) / `doubleWind`(연풍) |
| `kotsu`   | 커쯔·깡 배열. 각 원소는 `kind`(밍커/안커/밍깡/안깡) + `terminal`(노두·자패 여부)                  |

화면 전용 상태(`Calculator.tsx`): `mode`(score/fuOnly), `isDealer`(자/친), `han`(판수 1~13).

---

## 2. 모드

- **점수 계산하기(`score`)**: 자/친·판수를 추가로 받아 부수 + 점수를 계산.
- **부수만 계산하기(`fuOnly`)**: 부수만 계산. 자/친·판수·점수 표시 없음.
- 탭 전환 시 `<div className="calc-content" key={mode}>`의 `key`가 바뀌어 콘텐츠 전체가
  재마운트되며 진입 애니메이션이 다시 재생된다(이슈 #55).

---

## 3. 상태 간 제약 (UI 잠금·보정 규칙)

계산기의 핵심. 모순된 입력 조합이 만들어지지 않도록 파생 플래그로 입력을 잠그거나 보정한다.

### 3.1 입력 잠금

| 플래그          | 조건                                         | 효과                                                                                                                                  |
| --------------- | -------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| `fuLocked`      | `special !== 'none'`                         | 대기·또이쯔·커쯔 입력 비활성. 핑후/치또이는 부수가 고정이라 세부 항목이 무의미.                                                       |
| `fuIrrelevant`  | `mode === 'score' && han >= 5`               | 만관 이상은 부수가 점수에 무관 → 특수형태·대기·또이쯔·커쯔 전부 비활성, 부수 결과 표 숨김.                                            |
| `winFormLocked` | `special === 'chiitoi' && mode === 'fuOnly'` | 치또이 25부 고정이라 부수만 모드에선 화료 형태가 부수와 무관 → 화료 형태 잠금. 점수 모드에선 쯔모/론이 점수를 가르므로 잠그지 않는다. |
| 후로 론 비활성  | `special !== 'none'`                         | 화료 형태의 '후로 론' 옵션만 개별 비활성. 핑후·치또이는 멘젠 한정이라 후로 론과 양립 불가.                                            |
| 커쯔 추가 한도  | `input.kotsu.length >= 4`                    | 커쯔·깡은 최대 4개(4멘쯔)까지만 추가.                                                                                                 |

### 3.2 자동 보정

- **핑후/치또이를 켤 때 winForm 보정**(`toggleSpecial`): 후로 론 상태에서 핑후·치또이를
  켜면 멘젠 론으로 자동 전환한다(멘젠 한정 역이므로).
- **1판 불가 형태에서 판수 보정**: 아래 `hanFloorTwo` 참고.

### 3.3 판수 하한 (1판 불가) — `hanFloorTwo`

```
hanFloorTwo = special === 'chiitoi'
           || (special === 'pinfu' && winForm === 'tsumo')
```

- **치또이** = 2판 확정.
- **핑후 + 쯔모** = 핑후(1판) + 멘젠쯔모(1판)로 최소 2판. (핑후 + 론은 핑후 1판만으로 1판 성립 가능)
- 효과: 점수 모드에서 1판 버튼을 비활성화하고, 위 조건이 성립하는 순간 1판이 선택돼
  있었다면 2판으로 옮긴다.
- 진입 경로가 여럿이라(치또이 토글 / 핑후 토글 / 화료 형태를 쯔모로 변경) 각 핸들러
  (`toggleSpecial`, `setWinForm`)에서 진입 시점에 `setHan(2)`로 보정한다.

### 3.4 샤보 대기 경고 — `shanponNeedsKotsu`

```
shanponNeedsKotsu = special === 'none' && !fuIrrelevant
                 && wait === 'shanpon'
                 && 커쯔(밍커·안커)가 하나도 없음
```

샤보 대기는 또이쯔 하나가 커쯔로 완성되는 형태라 밍커·안커가 최소 1개 필요하다. 깡은 이미
4장으로 완성돼 대기 멘쯔가 될 수 없으므로 깡만 있으면 경고한다. 경고 중에는 결과를 숨긴다.

## 4. 부수 산식 — `calcFu` (`core/fu.ts`)

고정 부수 형태가 먼저 분기한다.

- **핑후 쯔모 = 20부 / 핑후 론(멘젠) = 30부** 고정.
- **치또이 = 25부** 고정.

일반형은 항목 합산:

| 항목      | 부수                               |
| --------- | ---------------------------------- |
| 부저      | 20                                 |
| 화료 형태 | 멘젠 론 +10 / 쯔모 +2 / 후로 론 +0 |
| 대기      | 간짱·변짱·단기 +2 (양면·샤보 0)    |
| 또이쯔    | 역패·장풍·자풍 +2 / 연풍 +4        |
| 커쯔·깡   | `kotsuFu` 결과 합 (아래)           |

`kotsuFu`: 기본 2부에서 `terminal`(노두·자패) ×2, `concealed`(안커·안깡) ×2, `quad`(깡) ×4.
→ 중장 밍커 2 / 안커 4 / 밍깡 8 / 안깡 16, 노두·자패는 각 ×2(밍커 4 … 안깡 32).

- **쿠이핑후형 보정**: 후로 론인데 합계가 20부면(가산이 부저뿐) 30부로 올린다.
- **10부 단위 절상**: 위 합계를 `roundUpFu`로 절상. 고정 부수(핑후·치또이)는 절상 대상 아님.

## 5. 점수 산식 — `computePoints` (`core/points.ts`)

- 기본점 `base = fu × 2^(2+han)`. 5판 이상은 부수와 무관한 한도 테이블로 분기.
- 한도: 5판 만관(2000) / 6~7판 하네만(3000) / 8~10판 배만(4000) / 11~12판 삼배만(6000) /
  13판↑ 헤아림역만(8000). 역만 복합은 `yakumanUnits`로 합산(2배역만 …).
- **키리아게 만관 미적용**(의도적 차이): `base`가 2000 이상으로 자연 도달할 때만 만관.
  4판 30부 등은 절상하지 않는다.
- 지불: 론 = `base ×(친6 / 자4)`, 친 쯔모 = `base×2` 올, 자 쯔모 = 자 `base` / 친 `base×2`.
  모두 100점 단위 절상(`roundUp100`).

## 6. 계산기가 다루지 않는 것 (스코프 경계)

- **역 판정·도라 계산 없음**: 판수는 사용자가 직접 입력한다. 따라서 "핑후인데 커쯔가 있다" 같은
  역-부수 모순은 입력 단계에서 잠금(`fuLocked`)으로 차단할 뿐, 판수와 부수의 정합성까지
  검증하지는 않는다.
- 본장·리치봉·후리텐·복수 화료 등은 다루지 않는다(점수 트레이너 v1 범위와 동일).
